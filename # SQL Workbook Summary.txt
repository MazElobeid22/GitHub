# SQL Workbook Summary

## Day 1: Database Foundations

### Task 1: Relationships and Keys
- **Primary Key**: unique, non-null
- **Secondary Key**: alternate unique ID
- **Foreign Key**: links tables, ensures integrity

### Connections
- One-to-One: student ↔ student ID
- One-to-Many: blog ↔ posts
- Many-to-Many: students ↔ classes

### Task 2: Relational vs Non-Relational
- **Relational (SQL)**: structured, schema-based
- **Non-Relational (NoSQL)**: flexible, schema-less (e.g., JSON, XML)

## Day 3: SQL Joins
- **Self-Join**: same table
- **Inner Join**: matching rows
- **Left/Right Join**: all from one side
- **Full Join**: all records
- **Cross Join**: Cartesian product

## Day 4: Database Design Essay
- Retail schema with 4 tables:
  - Products, Customers, Sales, Sale Items
- Relationships: customer ↔ sales, product ↔ supplier
- SQL commands: CREATE TABLE, INSERT INTO
- Maintenance: backups, access control

### Practical SQL Exercises
- Aggregation: city counts, population filters
- Pattern matching: cities starting with "New" or "Be"
- Sorting: top 10 populous cities
- Joins: countries ↔ cities
- Advanced: GDP ranks, density calculations
